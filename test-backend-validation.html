Issue: The validation middleware was using .isURL() which fails for empty strings.

Solution: Replaced with custom validation that allows empty strings OR valid URLs.

Before (causing the error):

body('socialLinks.twitter')
  .optional()
  .isURL()  // ‚ùå This fails for empty strings
  .withMessage('Twitter link must be a valid URL')

After (fixed):

body('socialLinks.twitter')
  .optional()
  .custom((value) => {
    // ‚úÖ Allow empty strings or valid URLs
    if (!value || value.trim() === '') return true;
    return /^https?:\/\/.+/.test(value);
  })
  .withMessage('Twitter link must be a valid URL or empty')
